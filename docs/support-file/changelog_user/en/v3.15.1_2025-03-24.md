### Feature

- [Feature] Support multi-tenant architecture, configurable to enable or disable multi-tenant mode; when enabled, supports system tenant and regular tenants; when disabled, uses a single default tenant
- [Feature] Support tenant data isolation, each tenant has independent data storage space to ensure business data does not interfere with each other
- [Feature] New tenant set management, system tenant can query all tenants, businesses and hosts across tenants through tenant sets for cross-tenant resource overview
- [Feature] Support automatic initialization for new tenants, automatically creating base models, model classifications, service categories, associations, resource pools and other initial data when adding a new tenant
- [Feature] Support tenant template mechanism, the system manages tenant base data uniformly through templates to ensure new tenants have complete initial configurations
- [Feature] Support global configuration management by tenant, each tenant can have independent global configuration
- [Feature] Support timezone feature, time fields in models are displayed in user's timezone, and Excel export supports user timezone conversion
- [Feature] Integration with BlueKing Audit Center, supporting audit data reporting
- [Feature] Field type supports plugin extension, new field types can be registered through plugins
- [Feature] Automatic frontend version update detection with refresh prompt
- [Feature] Added permission center and user management entry in the page
- [Feature] Provide migration tool for upgrading data from non-multi-tenant version to multi-tenant version, supporting smooth migration of existing data
